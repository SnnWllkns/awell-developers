---
title: Patient Demograhpics Query (PDQ)
description: Retrieve patient demographics from an external system
---

**When using Awell Care to operate your care flows, you might want to make it as easy as possible for your clinical personel to create patients in Awell.** The Patient Demographic Query (PDQ) integration allows your clinical stakeholders to retrieve patient demographic data from your system right within Awell. It only takes a press of a button to import the data from a patient in your system into Awell.

## Overview

The implementation is based on the [IHE ITI PDQm Profile](https://profiles.ihe.net/ITI/PDQm/index.html) although the full specification has not been implemented yet (see below).

## Actors

There are two actors in the PDQm profile:

1. **Patient Demographics Consumer** queries the patient based on available demographics (search parameters) and gets the complete set of patient demographics in return (if there are any).
2. **Patient Demographics Supplier** returns the complete set of available patient demographics based on the provided set of search parameters. The Founda Health Platform acts as the Patient Demographics Supplier in this profile.

## Request

A request will be made by the Patient Demographics Consumer (Awell) to solicit information about patients whose demographics data match data provided in the query parameters on the request message. The request can be made either using a `GET` or a `POST` request. Which method is used is defined in the developer settings in Awell.

### Search parameters

The request represents a [parameterized search](https://profiles.ihe.net/ITI/PDQm/ITI-78.html#23784121-search-parameters) from the Patient Demographics Consumer to the Patient Demographics Supplier. We currently only support searching on the `identifier` parameter. Which **coding system** and **code** that will be used for the search is configured in the developer settings in Awell.

**GET:**

```bash
curl {endpoint}/Patient?identifier:of-type={system}|{code}|{value}
```

**POST:**

```bash
curl {endpoint}/Patient/_search
  -X POST
  -H "Content-Type: application/x-www-form-urlencoded"
  -d "identifier:of-type={system}|{code}|{value}"
```

## Examples

### PDQ with national registry number

- Code system: http://terminology.hl7.org/CodeSystem/v2-0203
- Code: SB

```bash
curl {endpoint}/Patient?identifier:of-type=http://terminology.hl7.org/CodeSystem/v2-0203|SB|123456
```

### PDQ with visit number

- Code system: http://terminology.hl7.org/CodeSystem/v2-0203
- Code: VN

```bash
curl {endpoint}/Patient?identifier:of-type=http://terminology.hl7.org/CodeSystem/v2-0203|VN|123456
```

### PDQ with medical record number

- Code system: http://terminology.hl7.org/CodeSystem/v2-0203
- Code: MR

```bash
curl {endpoint}/Patient?identifier:of-type=http://terminology.hl7.org/CodeSystem/v2-0203|MR|123456
```
